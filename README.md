# Trenni

Supervisor for the [Yoitsu](https://github.com/morrejssc-hub/yoitsu) self-evolving agent system.

Trenni is event-driven: it receives events from Pasloe via webhook push (falling back to polling) and reacts. It owns job identity, concurrency, lifecycle, and evo version state. It does not execute agent logic — it launches Palimpsest as a subprocess per job and observes the resulting events.

Key boundaries:
- Job IDs are generated by Trenni, never by Palimpsest.
- Evo version selection (which commit is active) is Trenni's responsibility.
- Retry policy, escalation, and continuation decisions belong here, not in Palimpsest.

For system architecture and ADRs, see the [yoitsu repo](https://github.com/morrejssc-hub/yoitsu).

## Control API

Trenni exposes an HTTP control API (default: `http://127.0.0.1:8100`) co-hosted on the same event loop as the supervisor.

| Endpoint | Method | Description |
|---|---|---|
| `/control/status` | `GET` | Current supervisor state (running, paused, jobs, queue) |
| `/control/pause` | `POST` | Pause job dispatch (running jobs continue to completion) |
| `/control/resume` | `POST` | Resume job dispatch |
| `/control/stop` | `POST` | Graceful shutdown |
| `/hooks/events` | `POST` | Webhook intake from Pasloe |

CLI shortcuts:

```bash
trenni pause    # POST /control/pause
trenni resume   # POST /control/resume
```

## Webhook + Polling

On startup, Trenni registers itself as a webhook with Pasloe (`POST /webhooks`). Pasloe then pushes events in real-time as they are appended.

If webhook registration fails (e.g. Pasloe unreachable at startup), Trenni falls back to pure polling at `poll_interval`. When webhooks are active, polling continues at `webhook_poll_interval` (default 30 s) as a resilience fallback.

Webhook deliveries are HMAC-SHA256 signed when `webhook_secret` is set. On shutdown, the webhook is automatically deregistered.

## Configuration

```yaml
pasloe_url: "http://127.0.0.1:8000"
pasloe_api_key_env: "PASLOE_API_KEY"
source_id: "trenni-supervisor"

palimpsest_command: "palimpsest"
evo_repo_path: "./palimpsest-evo"
work_dir: "./trenni-work"

max_workers: 4
poll_interval: 2.0          # used when no webhook active

# Control API
api_host: "127.0.0.1"
api_port: 8100

# Webhook self-registration
webhook_secret: ""           # HMAC secret; empty = unsigned (fine on localhost)
trenni_public_url: ""        # Override if api_host:api_port is not reachable from Pasloe
webhook_poll_interval: 30.0  # Fallback poll interval when webhook active

# Job defaults (merged into every launched job)
default_llm:
  model: "claude-sonnet-4-6"
  api_key_env: "ANTHROPIC_API_KEY"
  max_iterations: 20
  max_tokens: 4096
default_workspace:
  git_token_env: "GITHUB_TOKEN"
```

## Running

```bash
uv run trenni start --config trenni-config.yaml
```
